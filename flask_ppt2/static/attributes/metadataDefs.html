Metadata Definitions

This document describes the metadata items and values used in the Project Portfolio Database. The usefulness of this database will depend on agreement about what these data mean, and our commitment to choosing carefully when editing the data.

The attributes are arranged into categories:

Those listed first, which make up the bulk of the data, simply describe the project in terms suitable for planning. The second category contains project portfolio data, used to compare projects against each other. The third category records a project's disposition at the end of every planning cycle: is it committed to the portfolio or is it deferred, to be reconsidered in a future cycle? Finally, there is some information about the project when it is actually being worked.
